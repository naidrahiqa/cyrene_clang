# CyreneClang ŌĆö Agent Guide

Custom LLVM/Clang toolchain for Android kernel builds. Tracks LLVM main, rebuilt weekly via GitHub Actions.

## Build

Single entry point: `bash scripts/build.sh`

Env vars that matter:

| Variable | Default | Notes |
|----------|---------|-------|
| `LLVM_BRANCH` | `llvmorg-22.1.0` | Must be a valid LLVM tag |
| `ENABLE_PGO` | `true` | 2-stage PGO build (slow, ~3h on CI) |
| `ENABLE_BOLT` | `true` | Post-build optimization, requires `perf` |
| `PGO_WORKLOAD` | `sqlite` | `kernel` is more accurate but clones android-mainline |
| `JOBS` | `$(nproc)` | CI uses 4; LLVM_PARALLEL_LINK_JOBS=1 hardcoded for ThinLTO safety |

Build is 2-stage when PGO enabled: stage1 = instrumented X86 build, profile collection, stage2 = AArch64 ThinLTO. When PGO disabled, single-pass `simple_build()`.

## CI quirks

- Runner: `ubuntu-latest` (Ubuntu 24.04+)
- **Package names**: use `libncurses-dev` (not `libncurses5-dev`), no `libgcc-*-dev` needed for Clang builds
- Disk is tight (~25GB usable after cleanup). Build scripts aggressively delete `.o` files and intermediate dirs during build. Do NOT remove cleanup logic.
- Tool detection must handle versioned names: `ld.lld-18`, `llvm-ar-18`, `llvm-ranlib-18` (Ubuntu 24.04)
- Workflow triggers on push to `patches/**` or `scripts/**`, plus `workflow_dispatch` and weekly schedule
- Build logs sent to Telegram via `scripts/notify.sh` (requires `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` secrets)

## Patches

- Location: `patches/*.patch` (format-patch style)
- Applied by `scripts/patch.sh` with 3-tier fallback: `git apply` ŌåÆ `--3way` ŌåÆ sed fallback for known issues
- Push to `patches/` auto-triggers CI build
- `patches/stable-picks.txt` lists LLVM stable commits for auto-sync (`sync-patches.yml`)

## Conventions

- Commits: `type: description` (`feat:`, `fix:`, `ci:`, `chore:`)
- All bash scripts start with `set -euo pipefail`
- Compression: `zstd` (not gzip)
- Never hardcode build flags in workflow YAML ŌĆö use `build.sh` env vars
- Toolchain installs to `$INSTALL_DIR` (default: `~/toolchains/cyrene`)
- Manifest: `clang-version.txt` + `clang_notes.txt` generated by `scripts/package.sh`

## Known Issues

- **Do NOT use `-DLLVM_USE_LINKER=lld` in cmake_configure()** ŌĆö it propagates to the runtimes sub-build where the just-built Clang fails the `-fuse-ld=lld` compiler check. Use standard CMake variables instead: `CMAKE_LINKER`, `CMAKE_EXE_LINKER_FLAGS`, `CMAKE_SHARED_LINKER_FLAGS`, `CMAKE_MODULE_LINKER_FLAGS`. See commit `2cd51af` for details.
